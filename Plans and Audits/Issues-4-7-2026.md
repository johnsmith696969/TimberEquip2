## HIGH Issues

### H-01: Privacy & DMCA contact sections hardcode dark background
**Files:** `src/pages/Privacy.tsx:587`, `src/pages/Dmca.tsx:379`
**Problem:** Both pages use `bg-[#1C1917] text-white` without theme switching. In light mode, these sections look like black boxes that don't match the rest of the page.
**Fix:** Apply the same pattern used on Contact.tsx — `theme === 'dark' ? 'bg-[#1C1917] text-white border border-white/10' : 'bg-surface text-ink border border-line'`

### H-02: Calculator results panel hardcodes dark background
**File:** `src/pages/Calculator.tsx:152`
**Problem:** `bg-[#1C1917] text-white` with no theme toggle. The loan results box is always dark regardless of theme.
**Fix:** Add theme-conditional className like other accent panels.

### H-03: Profile financing status badge hardcodes dark background
**File:** `src/pages/Profile.tsx:2164`
**Problem:** Financing request status badge uses `bg-[#1C1917] text-white` for the default/pending state. Looks like a dark blob in light mode.
**Fix:** Use `bg-surface text-ink border border-line` for light mode.

### H-04: 51 empty catch blocks across 27 files
**Problem:** Silent error swallowing in services and components. Key offenders:
- `src/services/placesService.ts` (5 instances)
- `src/components/AuthContext.tsx` (4 instances)
- `src/components/admin/CmsEditor.tsx` (4 instances)
- `src/services/userService.ts` (3 instances)
- `src/services/storageService.ts` (3 instances)
**Fix:** Add error logging or user-facing error feedback to each catch block.

### M-03: Auction pages use hardcoded semantic colors
**Files:** `AuctionDetail.tsx:56-88`, `Auctions.tsx:62-67`
**Problem:** Status badge colors use `bg-green-600`, `bg-red-600`, `bg-yellow-600` directly instead of theme tokens. These are acceptable for status indicators but won't adapt to custom themes if implemented later.
**Impact:** Low — semantic colors for status are industry standard.

### M-04: 227 instances of hardcoded `bg-white` / `text-white` across 33 page files
**Problem:** Many pages use `bg-white` and `text-white` instead of theme-aware classes. Not all are bugs — `text-white` on dark accent backgrounds is intentional. But some `bg-white` usages may break in dusk mode.
**Priority files to check:**
- `SellerProfile.tsx` (44 instances — highest count)
- `ListingDetail.tsx` (27 instances)
- `Unsubscribe.tsx` (13 instances)
- `AuctionDetail.tsx` (11 instances)
- `Home.tsx` (11 instances)
- `Categories.tsx` (10 instances)

### M-05: Auctions bid section `text-white/70` hardcoded in active auction cards
**File:** `Auctions.tsx:334`
**Problem:** `text-white/70` is hardcoded regardless of theme in the auction lot stats row. Fine if the parent is always dark, but could break if the card background changes with theme.

### M-06: DealerOS inventory table min-width may cause horizontal scroll issues
**File:** `DealerOS.tsx:1979`
**Problem:** Table has `min-w-[1100px]` which forces horizontal scroll on smaller screens. The table is in a scrollable container, but mobile experience suffers.
**Fix:** Consider responsive card layout for mobile, keeping table for desktop only.
